## How to start tomorrow morning

### 1. Bring the stack up (choose one)

**All-in-Docker (recommended):**
```bash
cd i:\DEVprojects\aLiveSpring\HydroFoil
npm run docker:up
# UI http://localhost:3000  |  API http://localhost:3001
# If port 3000 is busy (host Vite):  $env:ADMIN_UI_PORT='3010'; docker compose up -d --build
```

**Host dev (if you prefer Vite on host):**
```bash
npm run docker:up          # infra only — or stop app services in compose if you split
npm run migrate            # if DB fresh
npm run dev -w @hydrofoil/control-api
npm run dev -w @hydrofoil/media-worker
npm run dev -w @hydrofoil/admin-ui
```

**Gotcha:** only one process on **port 3000** (admin-ui). `strictPort: true` in Vite.

**Live preview / HLS in Docker:** UI proxies `/srs-media` and `/live` → SRS (`VITE_SRS_PLAYBACK_PROXY=http://srs:8080`). SRS master playlists use absolute `/live/...` URLs — without the `/live` proxy, Vite serves `index.html` and the player stays black. Direct check: `http://localhost:8080/live/<streamKey>.m3u8` → **200** while publishing; via UI: `http://localhost:3010/live/<streamKey>.m3u8` must return `#EXTM3U`, not HTML.

**WebRTC monitor:** SRS exposes WHEP on `:1985`; UI proxies `/srs-api` → SRS API. UDP **8000** must be published (`8000:8000/udp` in compose). If WHEP fails, Monitor falls back to HTTP-FLV automatically.

**SRT ingest:** UDP **10080** must be published (`10080:10080/udp` in compose). Publish with `srt://localhost:10080?streamid=#!::r=live/<stream-key>,m=publish` (copy from stream key settings). External SRT restream destinations start FFmpeg push on publish via media-worker.

### 2. Smoke-test routing (5 min)
1. Inputs → New Input → Generate stream key → Create  
2. Outputs → app `live`, stream name → Create  
3. Routes → link input → output(s)  
4. System Status → API/DB OK; gateway may show pending until worker runs  
5. Optional: publish to `rtmp://localhost:1935/live/<stream-key>`

### 3. Verify Docker build if compose was not run yet
```bash
docker compose up --build
```
First build can take several minutes (`npm ci` in image).
